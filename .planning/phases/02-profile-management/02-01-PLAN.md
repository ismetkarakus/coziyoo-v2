**Step 1: Create the `agent_profiles` table in Supabase.**

Apply this SQL directly to the database (do NOT create migration files -- per project convention):

```sql
CREATE TABLE agent_profiles (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name          TEXT NOT NULL,
  is_active     BOOLEAN NOT NULL DEFAULT FALSE,
  speaks_first  BOOLEAN NOT NULL DEFAULT FALSE,
  system_prompt TEXT,
  greeting_enabled BOOLEAN NOT NULL DEFAULT TRUE,
  greeting_instruction TEXT,
  voice_language TEXT NOT NULL DEFAULT 'tr',
  llm_config    JSONB NOT NULL DEFAULT '{}',
  stt_config    JSONB NOT NULL DEFAULT '{}',
  tts_config    JSONB NOT NULL DEFAULT '{}',
  n8n_config    JSONB NOT NULL DEFAULT '{}',
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE UNIQUE INDEX agent_profiles_one_active_idx
  ON agent_profiles (is_active) WHERE is_active = TRUE;

CREATE OR REPLACE FUNCTION update_agent_profiles_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER agent_profiles_updated_at
  BEFORE UPDATE ON agent_profiles
  FOR EACH ROW EXECUTE FUNCTION update_agent_profiles_updated_at();
```

Note: `speaks_first` (MODEL-06) controls whether the assistant initiates the conversation with a first message (true) or waits for the user to speak (false).

**Step 2: Create `apps/api/src/routes/admin-agent-profiles.ts`.**

Create a new Express router file with these 7 endpoints, all protected by `requireAuth("admin")`:

1. **GET `/`** -- List all profiles, ordered by `is_active DESC, updated_at DESC`. Return `{ data: AgentProfile[] }`.

2. **POST `/`** -- Create a profile. Accept body: `{ name: string, speaks_first?: boolean, system_prompt?: string, greeting_enabled?: boolean, greeting_instruction?: string, voice_language?: string, llm_config?: object, stt_config?: object, tts_config?: object, n8n_config?: object }`. Validate `name` is non-empty string (max 128 chars). INSERT and return `{ data: AgentProfile }` with 201 status.

3. **GET `/:id`** -- Get profile by UUID. Return `{ data: AgentProfile }` or 404.

4. **PUT `/:id`** -- Update profile. Accept partial body (any subset of create fields). Use SQL `COALESCE` pattern to only update provided fields:
   ```sql
   UPDATE agent_profiles SET
     name = COALESCE($2, name),
     speaks_first = COALESCE($3, speaks_first),
     system_prompt = COALESCE($4, system_prompt),
     greeting_enabled = COALESCE($5, greeting_enabled),
     greeting_instruction = COALESCE($6, greeting_instruction),
     voice_language = COALESCE($7, voice_language),
     llm_config = COALESCE($8, llm_config),
     stt_config = COALESCE($9, stt_config),
     tts_config = COALESCE($10, tts_config),
     n8n_config = COALESCE($11, n8n_config)
   WHERE id = $1
   RETURNING *
   ```
   Return `{ data: AgentProfile }` or 404.

5. **DELETE `/:id`** -- Delete profile. FIRST check if `is_active = TRUE` for this profile. If active, return 409 `{ error: { code: "CANNOT_DELETE_ACTIVE", message: "Cannot delete the active profile. Activate a different profile first." } }`. Otherwise DELETE and return `{ data: { deleted: id } }`.

6. **POST `/:id/activate`** -- Activate profile exclusively. Use transaction:
   ```sql
   BEGIN;
   UPDATE agent_profiles SET is_active = FALSE WHERE is_active = TRUE;
   UPDATE agent_profiles SET is_active = TRUE WHERE id = $1 RETURNING id;
   -- if rowCount === 0, ROLLBACK and return 404
   COMMIT;
   ```
   Return `{ data: { active: id } }`.

7. **POST `/:id/duplicate`** -- Clone profile. SELECT the source profile by id, INSERT a new row with `name = source.name + ' (copy)'` and all config columns copied (including `speaks_first`). Return `{ data: AgentProfile }` with 201.

Export: `export const agentProfilesRouter = router;`

Use the same error handling pattern as `admin-livekit.ts`: try/catch with `{ error: { code, message } }` responses.
