# Plan 04-01: Make AI_SERVER_SHARED_SECRET Required

## Goal

Both the API and voice agent fail fast at startup with a clear error message when `AI_SERVER_SHARED_SECRET` is missing or shorter than 16 characters, instead of silently accepting an empty string.

## Context

`AI_SERVER_SHARED_SECRET` is the shared secret that authenticates calls between the API and the voice agent. Currently both sides accept its absence silently:

- API (`apps/api/src/config/env.ts` line 39): declared as `z.string().min(16).optional()`. Startup Zod parse succeeds without it. The secret is only checked per-route at request time, returning HTTP 503.
- Voice agent (`apps/voice-agent/src/voice_agent/config/settings.py` line 27): defaults to `""` via `os.getenv("AI_SERVER_SHARED_SECRET", "")`. No startup check.
- `join_api.py` line 451: raises HTTP 503 at request time if empty, but the process does not exit.

The fix is purely making existing guardrails trigger at startup instead of at first request.

## Requirements Addressed

| Req ID | Description |
|--------|-------------|
| TURN-05 | AI_SERVER_SHARED_SECRET is required (not optional) and validated at startup on both API and voice agent; missing secret fails fast with a clear error |

## Tasks

### Wave 1 — Make secret required in both processes

> Prerequisite: None. Both changes are independent and can be applied together.

#### Task 04-01-01: Remove `.optional()` from API env schema

**File:** `apps/api/src/config/env.ts`

**What:** Change line 39 from `z.string().min(16).optional()` to `z.string().min(16)`.

**Why:** The existing `safeParse` call at line 93 already calls `process.exit(1)` on parse failure with `console.error("Invalid environment variables", parsed.error.flatten().fieldErrors)`. Removing `.optional()` is the only change needed — startup will refuse to continue if the secret is absent or shorter than 16 chars.

**How:**
```typescript
// Before (line 39):
AI_SERVER_SHARED_SECRET: z.string().min(16).optional(),

// After:
AI_SERVER_SHARED_SECRET: z.string().min(16),
```

No other changes to `env.ts` are needed. The TypeScript type of `env.AI_SERVER_SHARED_SECRET` will change from `string | undefined` to `string`. Downstream code that guarded against `undefined` (the three per-route `if (!env.AI_SERVER_SHARED_SECRET)` blocks in `apps/api/src/routes/livekit.ts`) will now be dead code — leave them in place as defense-in-depth; TypeScript may emit a "always truthy" narrowing warning but it will not break the build.

**Verify:**
```bash
# Type-check only (fast)
npm run build:api 2>&1 | grep -E "error|AI_SERVER_SHARED_SECRET"
# Confirm parse failure without the secret:
AI_SERVER_SHARED_SECRET="" node -e "require('./apps/api/dist/src/config/env.js')" 2>&1 | grep -i "invalid\|secret" || echo "set the real secret to test absence"
```
Manual: Remove `AI_SERVER_SHARED_SECRET` from `.env`, start the API with `npm run dev:api`, confirm it exits immediately with an "Invalid environment variables" message listing `AI_SERVER_SHARED_SECRET`.

#### Task 04-01-02: Add startup validation in voice agent entrypoint and join API

**Files:**
- `apps/voice-agent/src/voice_agent/entrypoint.py`
- `apps/voice-agent/src/voice_agent/join_api.py`

**What:** In `entrypoint.py`, add a guard inside `main()` (line 1150) after `_configure_logging()`. In `join_api.py`, add a module-level guard executed at import time (before `app = FastAPI(...)`, after `settings = get_settings()` on line 17).

**Why:** The two Python processes (worker and join API) start independently. Both need to fail fast because an empty secret means no call between the API and voice agent can be authenticated.

**How:**

In `entrypoint.py`, replace the `main()` function body:
```python
def main() -> None:
    _configure_logging()
    # Validate required secrets before doing anything else
    if not settings.ai_server_shared_secret or len(settings.ai_server_shared_secret) < 16:
        logger.critical(
            "AI_SERVER_SHARED_SECRET is required and must be at least 16 characters. "
            "Set it in .env or the environment before starting the voice agent worker."
        )
        sys.exit(1)
    _start_worker_heartbeat()
    cli.run_app(server)
```

`sys` is already imported (check imports at top of entrypoint.py; add `import sys` if missing).

In `join_api.py`, insert after line 17 (`settings = get_settings()`):
```python
# Fail fast at startup — do not serve requests with a missing or weak secret
if not settings.ai_server_shared_secret or len(settings.ai_server_shared_secret) < 16:
    raise RuntimeError(
        "AI_SERVER_SHARED_SECRET is required and must be at least 16 characters. "
        "Set it in .env or the environment before starting the join API."
    )
```
A `RuntimeError` at module scope will cause uvicorn to refuse to start with a clear traceback.

**Verify:**
```bash

cd /Users/drascom/Work/coziyoo-v2/apps/voice-agent
AI_SERVER_SHARED_SECRET="" python -c "from voice_agent.entrypoint import main; main()" 2>&1 | grep -i "required\|SHARED_SECRET"

# Unset the secret and try to import join_api (should raise RuntimeError)
AI_SERVER_SHARED_SECRET="" python -c "import voice_agent.join_api" 2>&1 | grep -i "required\|SHARED_SECRET"
```

## Risks

- **Existing deployments without the secret set will stop starting after this change.** Before deploying, operators must ensure `AI_SERVER_SHARED_SECRET` is in their `.env`. The `.env.example` update in Task 3 documents this. Add a deploy note in the PR description.
- **TypeScript narrowing warnings:** Downstream guards like `if (!env.AI_SERVER_SHARED_SECRET)` become redundant after the type changes from `string | undefined` to `string`. The TypeScript compiler may warn but this is not a build error in strict mode — it is a narrowing of a truthy type.
