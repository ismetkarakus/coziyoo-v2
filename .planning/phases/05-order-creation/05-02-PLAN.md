## PART A: Add add_to_cart → session_memory write

The brain workflow currently sends a UI action on add_to_cart but does NOT persist to Supabase. Fix this first.

### Step A1: Add "Write Cart to Memory" Postgres node

In the n8n UI, open workflow **brain_6KFFgjd26nF0kNCA** (Brain Workflow).

After the **Needs MCP? → true → RAG Lookup → Call MCP Gateway** path, find where add_to_cart MCP response returns. Add a new node between "Call MCP Gateway" and "Merge Context":

- **Node type:** Postgres (use the "Supabase Postgres" credential from Plan 05-01)
- **Node name:** `Write Cart to Memory`
- **Operation:** Execute Query
- **Query:**
```sql
INSERT INTO session_memory (room_id, user_id, data, created_at, updated_at)
VALUES (
  '{{ $('Normalize Input').item.json.roomId }}',
  '{{ $('Normalize Input').item.json.userId }}',
  '{{ JSON.stringify({ cart: $json.cart || [], sellerId: $json.sellerId, userId: $('Normalize Input').item.json.userId }) }}'::jsonb,
  now(),
  now()
)
ON CONFLICT (room_id)
DO UPDATE SET
  data = session_memory.data || EXCLUDED.data,
  updated_at = now();
```

BUT: the cart accumulation logic must be done in a Code node BEFORE this SQL because n8n expressions cannot call JSON.stringify on complex objects inline. Instead:

**Actually do this:**

1. Add a **Code node** named `Build Cart Upsert` AFTER "Call MCP Gateway", BEFORE "Write Cart to Memory":
```javascript
// Build the incremental cart upsert from MCP response + current turn slots
const prev = $('Normalize Input').item.json;
const mcpResult = $json; // output from Call MCP Gateway
const slots = prev.slots || {};

// Extract the product returned by MCP (products.search result)
const mcpData = mcpResult?.data || {};
const foods = mcpData.foods || mcpData.results || [];
const match = foods[0] || null;

if (!match || !match.lotId) {
  // No lot found — skip cart write, continue to merge
  return [{ json: { ...prev, cartWriteSkipped: true, cartWriteReason: 'no_lot_found' } }];
}

const newItem = {
  lotId: match.lotId,
  foodName: match.name || match.foodName || slots.productName || 'Unknown',
  quantity: slots.quantity || 1,
  unitPrice: parseFloat(match.price) || 0,
  sellerId: match.sellerId || prev.sellerId || null,
};

return [{ json: {
  ...prev,
  cartUpsertPayload: {
    newItem,
    roomId: prev.roomId,
    userId: prev.userId,
  }
}}];
```

2. Connect: Call MCP Gateway → Build Cart Upsert → Write Cart to Memory → Merge Context

For Write Cart to Memory Postgres node, use a Code node approach instead — n8n Postgres "Execute Query" with parameters:

- **Query:**
```sql
INSERT INTO session_memory (room_id, user_id, data, created_at, updated_at)
VALUES ($1, $2, $3::jsonb, now(), now())
ON CONFLICT (room_id)
DO UPDATE SET
  data = jsonb_set(
    COALESCE(session_memory.data, '{}'::jsonb),
    '{cart}',
    COALESCE(session_memory.data->'cart', '[]'::jsonb) || $4::jsonb,
    true
  ) || jsonb_build_object('userId', $2::text, 'sellerId', $5::text),
  updated_at = now()
```
- **Query Parameters:**
  - $1 = `{{ $json.cartUpsertPayload.roomId }}`
  - $2 = `{{ $json.cartUpsertPayload.userId }}`
  - $3 = `{{ JSON.stringify({ cart: [$json.cartUpsertPayload.newItem], userId: $json.cartUpsertPayload.userId, sellerId: $json.cartUpsertPayload.newItem.sellerId }) }}`
  - $4 = `{{ JSON.stringify([$json.cartUpsertPayload.newItem]) }}`
  - $5 = `{{ $json.cartUpsertPayload.newItem.sellerId }}`

**Simpler alternative (recommended for v1):** Skip the complex jsonb merge in SQL. Instead, in the Code node, read the existing cart from session_memory FIRST, append the new item, then do a full REPLACE upsert:

**Recommended approach — Replace full data on each add_to_cart:**

1. Add **"Read Session Memory"** Postgres node (Execute Query):
```sql
SELECT data FROM session_memory WHERE room_id = $1
```
Parameter: `{{ $('Normalize Input').item.json.roomId }}`

2. Add **"Append to Cart"** Code node:
```javascript
const prev = $('Normalize Input').item.json;
const mcpResult = $('Call MCP Gateway').item.json;
const existingData = $json.data || {};  // from Read Session Memory
const existingCart = existingData.cart || [];

const foods = mcpResult?.data?.foods || mcpResult?.data?.results || [];
const match = foods[0] || null;
if (!match || !match.lotId) {
  return [{ json: { ...prev, cartWriteSkipped: true } }];
}

const slots = prev.slots || {};
const newItem = {
  lotId: match.lotId,
  foodName: match.name || slots.productName || 'Unknown',
  quantity: typeof slots.quantity === 'number' ? slots.quantity : 1,
  unitPrice: parseFloat(match.price) || 0,
};

// Append (replace same lotId if already in cart)
const cart = [...existingCart.filter(i => i.lotId !== newItem.lotId), newItem];

return [{ json: {
  ...prev,
  updatedCart: cart,
  updatedSellerId: match.sellerId || existingData.sellerId || null,
  updatedUserId: prev.userId || null,
}}];
```

3. Add **"Write Session Memory"** Postgres node (Execute Query):
```sql
INSERT INTO session_memory (room_id, user_id, data, created_at, updated_at)
VALUES ($1, $2::uuid, $3::jsonb, now(), now())
ON CONFLICT (room_id)
DO UPDATE SET data = $3::jsonb, updated_at = now()
```
Parameters:
- $1 = `{{ $json.roomId }}`
- $2 = `{{ $json.updatedUserId }}`
- $3 = `{{ JSON.stringify({ cart: $json.updatedCart, sellerId: $json.updatedSellerId, userId: $json.updatedUserId }) }}`

Wire: Call MCP Gateway → Read Session Memory → Append to Cart → Write Session Memory → Merge Context

## PART B: Add checkout → order creation branch

### Step B1: Add "Is Checkout?" IF node

After **Parse Intent**, the flow goes to **Needs MCP?**. The checkout intent has requiresMcp = false, so it exits the false branch of Needs MCP? and goes to Merge Context.

**Add a new IF node before Merge Context on the false branch:**

- Place **"Is Checkout?"** IF node between "Needs MCP? [false output]" and "Merge Context"
- Condition: `{{ $json.intent }}` equals `checkout`
- **False branch** → Merge Context (existing behavior for all non-checkout, non-MCP intents)
- **True branch** → new checkout sub-graph (steps below)

### Step B2: Add "Read Cart for Checkout" Postgres node

- **Node type:** Postgres ("Supabase Postgres" credential)
- **Node name:** `Read Cart for Checkout`
- **Operation:** Execute Query
- **Query:** `SELECT data FROM session_memory WHERE room_id = $1`
- **Parameter $1:** `{{ $('Parse Intent').item.json.roomId }}`

### Step B3: Add "Build Order Payload" Code node

- **Node name:** `Build Order Payload`
- **Code:**
```javascript
const sessionData = $json.data || {};
const cart = sessionData.cart || [];
const userId = sessionData.userId || null;
const sellerId = sessionData.sellerId || null;
const prev = $('Parse Intent').item.json;

if (cart.length === 0 || !userId || !sellerId) {
  return [{ json: {
    ...prev,
    orderError: true,
    orderErrorReason: !cart.length ? 'empty_cart' : !userId ? 'missing_userId' : 'missing_sellerId',
    replyText: cart.length === 0
      ? "I don't see any items in your cart yet. Would you like to add something?"
      : "I couldn't identify your account. Please restart the session.",
  }}];
}

const items = cart.map(item => ({ lotId: item.lotId, quantity: item.quantity || 1 }));
const idempotencyKey = `${prev.roomId}-order-${Date.now()}`;

return [{ json: {
  ...prev,
  orderPayload: { userId, sellerId, deliveryType: 'pickup', items },
  idempotencyKey,
  orderError: false,
}}];
```

### Step B4: Add "Has Cart?" IF node

- **Node name:** `Has Cart?`
- **Condition:** `{{ $json.orderError }}` is `false` (boolean false)
- **True branch** (has cart) → POST /v1/orders/voice HTTP Request node
- **False branch** (no cart or missing data) → Checkout Error Reply node

### Step B5: Add "POST Create Voice Order" HTTP Request node

- **Node name:** `POST Create Voice Order`
- **Method:** POST
- **URL:** `http://127.0.0.1:3000/v1/orders/voice`
- **Send Headers:** YES
  - `Content-Type`: `application/json`
  - `x-ai-server-secret`: `{{ $env.AI_SERVER_SHARED_SECRET }}`
  - `Idempotency-Key`: `{{ $json.idempotencyKey }}`
- **Send Body:** JSON
  - Body: `{{ JSON.stringify($json.orderPayload) }}`
- **Response Format:** JSON
- **On Error:** Continue (do not throw — handle in next node)

### Step B6: Add "Handle Order Response" Code node

- **Node name:** `Handle Order Response`
- **Code:**
```javascript
const httpResponse = $json;
const prev = $('Build Order Payload').item.json;

// Check for HTTP error (n8n wraps error responses)
const status = httpResponse.$response?.statusCode || httpResponse.statusCode || 201;
const body = httpResponse.data || httpResponse.body || httpResponse;

if (status >= 400 || httpResponse.error) {
  const errCode = httpResponse.error?.code || body?.error?.code || 'UNKNOWN_ERROR';
  const errMsg = httpResponse.error?.message || body?.error?.message || 'Order could not be created.';
  return [{ json: {
    ...prev,
    replyText: `Sorry, I couldn't place the order. ${errMsg}`,
    actions: [],
    orderSuccess: false,
    orderError: true,
    orderErrorCode: errCode,
  }}];
}

const orderId = body?.data?.orderId || body?.orderId;
const totalPrice = body?.data?.totalPrice || body?.totalPrice || 0;

return [{ json: {
  ...prev,
  replyText: `Your order has been placed! Order ID is ${orderId}. Total: ${totalPrice}. The cook has been notified.`,
  actions: [{ name: 'navigate', params: { screen: 'OrderDetail', orderId } }],
  orderSuccess: true,
  orderId,
  totalPrice,
}}];
```

### Step B7: Add "Checkout Error Reply" Code node (false branch of Has Cart?)

- **Node name:** `Checkout Error Reply`
- **Code:**
```javascript
const prev = $json;
return [{ json: {
  ...prev,
  replyText: prev.replyText || "I couldn't complete the order. Please try again.",
  actions: [],
}}];
```

### Step B8: Wire checkout branch into Respond node

Both "Handle Order Response" and "Checkout Error Reply" must connect to a new **"Checkout Finalize"** Code node that formats the final respond shape, then connects to **Respond**:

- **Node name:** `Checkout Finalize`
- **Code:**
```javascript
// Shape matches what voice agent expects: { replyText, actions }
const ctx = $json;
return [{ json: {
  replyText: ctx.replyText || 'Order processed.',
  actions: ctx.actions || [],
  meta: {
    intent: 'checkout',
    orderSuccess: ctx.orderSuccess || false,
    orderId: ctx.orderId || null,
  }
}}];
```

Connect: Checkout Finalize → Respond (same Respond node the main flow uses).

**IMPORTANT:** The Respond node responds to the original webhook. Only ONE path can reach it per execution. The checkout branch REPLACES the Reply LLM path entirely — it does not call the LLM at all. This is correct because the order confirmation message is deterministic.
