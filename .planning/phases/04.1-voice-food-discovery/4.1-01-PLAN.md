## STEP 7: Add to Cart Branch (output 1)

### Node: "Search Food for Cart" (HTTP Request — same as Search Foods but limit=3)
- Method: GET
- URL: `http://127.0.0.1:3000/v1/voice/foods`
- Query Parameters:
  - `q`: `{{ $json.slots?.productName || '' }}`
  - `limit`: `3`
- Headers: `x-ai-server-secret: {{ $env.AI_SERVER_SHARED_SECRET }}`

### Node: "Write Cart to Memory" (Postgres — Supabase Postgres credential)
- Operation: Execute Query
- Query:
```sql
INSERT INTO session_memory (room_id, user_id, data, created_at, updated_at)
VALUES ($1, $2::uuid, $3::jsonb, now(), now())
ON CONFLICT (room_id) DO UPDATE
SET data = jsonb_set(
      COALESCE(session_memory.data, '{}'::jsonb) || jsonb_build_object('userId', $2::text),
      '{cart}',
      COALESCE(
        (SELECT jsonb_agg(item) FROM jsonb_array_elements(
          COALESCE(session_memory.data->'cart', '[]'::jsonb)
        ) item WHERE item->>'lotId' != $4::text),
        '[]'::jsonb
      ) || $5::jsonb,
      true
    ),
    updated_at = now()
```
- Parameters:
  - $1: `{{ $('Parse Intent').item.json.roomId }}`
  - $2: `{{ $('Parse Intent').item.json.userId }}`
  - $3: `{{ JSON.stringify({ cart: [], userId: $('Parse Intent').item.json.userId }) }}`
  - $4: `{{ ($json.data?.foods?.find(f => f.available && f.lotId))?.lotId || '' }}`
  - $5: `{{ JSON.stringify([{ lotId: ($json.data?.foods?.find(f => f.available && f.lotId))?.lotId, foodName: ($json.data?.foods?.find(f => f.available && f.lotId))?.name, quantity: $('Parse Intent').item.json.slots?.quantity || 1, unitPrice: parseFloat(($json.data?.foods?.find(f => f.available && f.lotId))?.price) || 0 }]) }}`

**Simpler alternative** if the Postgres node expressions are too complex — use a Code node before the Postgres node:

### Alternative: Add "Prepare Cart Item" Code node before "Write Cart to Memory"
```javascript
const prev = $('Parse Intent').item.json;
```

```javascript
  contextForReply: `The user wants to add to cart. Successfully added: ${prev.cartItem.foodName} x${prev.cartItem.quantity} at ${prev.cartItem.unitPrice} TL. Confirm this to the user naturally in ${prev.locale === 'tr' ? 'Turkish' : 'English'}.`,
```

## STEP 8: Checkout Branch (output 2)

### Node: "Read Cart" (Postgres)
- Query: `SELECT data FROM session_memory WHERE room_id = $1`
- $1: `{{ $('Parse Intent').item.json.roomId }}`

### Node: "Prepare Order" (Code — ~15 lines, unavoidable logic)
```javascript
const prev = $('Parse Intent').item.json;
const sessionData = $json.data || {};
const cart = sessionData.cart || [];
const userId = sessionData.userId || prev.userId || null;
const sellerId = cart[0]?.sellerId || null;

if (cart.length === 0 || !userId) {
  return [{ json: { ...prev, orderError: true, contextForReply: `The cart is empty. Tell the user to add items first. Respond in ${prev.locale === 'tr' ? 'Turkish' : 'English'}.`, conversationHistory: prev.messages } }];
}

return [{ json: {
  ...prev,
  orderPayload: { userId, sellerId: sellerId || '', deliveryType: 'pickup', items: cart.map(i => ({ lotId: i.lotId, quantity: i.quantity || 1 })) },
  idempotencyKey: `${prev.roomId}-order-${Date.now()}`,
  orderError: false,
  conversationHistory: prev.messages,
}}];
```

### Node: "Has Cart?" (IF)
- Condition: `{{ $json.orderError }}` equals `false`
- True → Create Order
- False → Reply LLM (with contextForReply already set)

### Node: "Create Order" (HTTP Request)
- Method: POST, URL: `http://127.0.0.1:3000/v1/orders/voice`
- Headers: `x-ai-server-secret: {{ $env.AI_SERVER_SHARED_SECRET }}`, `Content-Type: application/json`, `Idempotency-Key: {{ $json.idempotencyKey }}`
- Body: `{{ JSON.stringify($json.orderPayload) }}`
- On Error: Continue

### Node: "Notify Cook" (HTTP Request)
- Method: POST
- URL: `http://127.0.0.1:3000/v1/orders/{{ $json.data?.orderId }}/notify-cook`
- Headers: `x-ai-server-secret: {{ $env.AI_SERVER_SHARED_SECRET }}`, `Content-Type: application/json`
- On Error: Continue

### Node: "Build Order Reply Context" (Code — minimal)
```javascript
const prev = $('Prepare Order').item.json;
const orderResult = $('Create Order').item.json;
const orderId = orderResult?.data?.orderId;
const total = orderResult?.data?.totalPrice || 0;
const success = !!orderId;
return [{ json: {
  ...prev,
  contextForReply: success
    ? `Order placed successfully! Order ID: ${orderId}, Total: ${total} TL, cook has been notified. Confirm to user in ${prev.locale === 'tr' ? 'Turkish' : 'English'}.`
    : `Order failed: ${orderResult?.error?.message || 'unknown error'}. Apologize and tell user to try again in ${prev.locale === 'tr' ? 'Turkish' : 'English'}.`,
  actions: success ? [{ name: 'navigate', params: { screen: 'OrderDetail', orderId } }] : [],
  conversationHistory: prev.messages,
}}];
```

**Option A (recommended): Duplicate Reply LLM node** — one per branch. Each branch has its own Reply LLM → Respond chain. This is clean in n8n.

```
You are Coziyoo, a friendly voice food ordering assistant.
Given the context below, generate a natural spoken response.
Output ONLY valid JSON: {"replyText":"<your response>","actions":[]}
Keep responses short (1-3 sentences) — this is voice output.
```

**User message field:** `{{ $json.contextForReply }}`

```javascript
const raw = $input.first().json.message?.content || $input.first().json.choices?.[0]?.message?.content || $input.first().json.response || '{}';
```
